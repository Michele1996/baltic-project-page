BALTIC

A Benchmark and Cross-Domain Strategy for 3D Reconstruction Across Air and Underwater Domains Under Varying Illumination

Cross-domain reconstruction: underwater + in-air views significantly improve robustness and completeness.

Overview

BALTIC is a controlled benchmark for evaluating 3D reconstruction methods across air and underwater environments under varying illumination. It enables systematic analysis of scattering, absorption, and domain shifts.

Devkit

We provide a lightweight evaluation toolkit for assessing 3D reconstruction quality and trajectory accuracy on the BALTIC dataset. The devkit operates on COLMAP reconstructions and ground-truth tracking data, enabling consistent and reproducible evaluation across all sequences. 

git clone https://github.com/Michele1996/baltic-project-page.git
cd baltic-project-page

# 1. Run COLMAP reconstruction externally (not included)

# 2. Geometry evaluation (Chamfer, roughness, NN spacing)
python chamfer.py

# 3. Trajectory alignment + metrics (ATE, RPE)
python alignment.py

The evaluation pipeline computes geometric metrics (Chamfer Distance, surface roughness, nearest-neighbor spacing) and trajectory metrics (Absolute Trajectory Error, Relative Pose Error) after Umeyama alignment.

Data Format

Each sequence contains reconstructed and ground-truth trajectories stored as text files.

  • E*_xyz_q.txt — reconstructed trajectory from COLMAP
  • E*_tf.txt — ground-truth trajectory from tracking system

Reconstructed trajectory format:

qw qx qy qz x y z

Quaternion (qw, qx, qy, qz) represents camera orientation, and (x, y, z) represents camera position in 3D space.

Ground-truth trajectory format:

timestamp frame | Translation: x=..., y=..., z=... | Rotation: x=..., y=..., z=..., w=...

Each entry contains a timestamped rigid transformation. Only the translation component is used for trajectory evaluation, while rotation is provided for completeness.

Evaluation Procedure

  1. Load reconstructed COLMAP trajectory (E*_xyz_q.txt)
  2. Load ground-truth trajectory (E*_tf.txt)
  3. Align trajectories using similarity transform (Umeyama)
  4. Compute ATE and RPE metrics
  5. Evaluate geometric quality from COLMAP point clouds

Benchmark Protocol

BALTIC is designed to evaluate the robustness of 3D reconstruction methods under strong domain shifts between air and underwater environments.

  • Task: Structure-from-Motion / Multi-view 3D reconstruction
  • Domains: Air, underwater, and mixed illumination conditions
  • Evaluation focus: Geometry quality and trajectory consistency

All methods are evaluated using a common COLMAP-based reconstruction pipeline followed by geometric and trajectory analysis.

Qualitative Results

Example COLMAP reconstructions across air and underwater conditions. Air sequences produce dense and globally consistent point clouds, while underwater scenes exhibit reduced density and fragmentation due to scattering and absorption effects.

Reproducibility

All results can be reproduced using standard COLMAP reconstruction followed by the provided evaluation toolkit.

  • Run COLMAP on each sequence (E1–E13)
  • Compute geometry metrics using chamfer.py
  • Compute trajectory metrics using alignment.py

The full dataset and evaluation scripts are publicly available for reproducibility.
